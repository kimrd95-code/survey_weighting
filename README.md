# Взвешивание опросов (Streamlit)

Два режима в боковой панели:

1. **Внешние цели (Росстат)** — постстратификация по матрице/таблице численностей: вес в ячейке = целевая доля ÷ фактическая; нормировка к среднему 1 **не применяется** (сумма весов ≈ N при полном покрытии).
2. **Группа на группу** — внутренняя калибровка: соц-дем структура взвешиваемой группы подгоняется под целевую; у **целевой группы (эталона) вес фиксированно 1**, взвешиваемая группа масштабируется относительно неё.

В обоих режимах в Excel добавляется столбец весов: в **первой** строке над заголовком — код **WT**, во **второй** (строка имён колонок) — подпись **веса** (как у опроса с двухстрочным заголовком).

## Логика (режим Росстат)

- **Взвешенное** распределение по ячейкам совпадает с целевым (с учётом исключений в файле Росстата).

## Развёртывание на Streamlit Community Cloud

1. Репозиторий на GitHub с `app.py` и `requirements.txt`.
2. [share.streamlit.io](https://share.streamlit.io) → подключить репозиторий.
3. **Main file path:** `app.py`.

Локально: Python **3.10+**.

## Установка и запуск

```powershell
cd survey_weighting_app
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
streamlit run app.py
```

## Данные

- Строка с именами переменных в опросе задаётся в боковой панели (часто это **2-я** строка листа).
- Файл Росстата: **матрица** (строки × столбцы с численностями) или **таблица** с измерениями и столбцом численности/доли.
- При скачивании в исходный Excel добавляется столбец: `WT` в строке над заголовком, `веса` в строке с названиями колонок (при однострочном заголовке — только `веса`). Запись через openpyxl сохраняет остальные ячейки листа.
